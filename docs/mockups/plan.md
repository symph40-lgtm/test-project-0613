# Mockup Plan

## Overview

스탁가드 v1의 프론트엔드 목업 스프린트 계획이다. 코드를 작성하기 전에, PRD의 6개 사용자 시나리오와 userflow의 5개 플로우를 "사용자가 눈으로 리뷰할 수 있는 화면 단위"로 분해한다. 목표는 더미 데이터 기반의 정적·상태별 화면을 그려, 정보 우선순위·행동 결론 표현·톤이 브랜드 의도(차분한 리스크 코치)와 맞는지 화면 단위로 확정하는 것이다.

이번 문서는 v1 범위의 18개 화면을 다룬다. 승인 게이트(이용 신청·관리자 승인)부터 아침 브리핑·포지션·장중 알림·마감 전 판단·행동 기록까지의 핵심 가치 루프를 포함한다.

## Source Context

- PRD: 승인 기반 게이트 접근(FR-030~036), 결론 우선 아침 브리핑(FR-002~005), 9단계 장세·행동 라벨(FR-003~004), 포지션·종목별 위험도(FR-007~008), 추천 위험선·개인 원칙(FR-009~010), 장중 원칙 기반 알림·미준수 리스크(FR-012~016), 장중 체결 재판단(FR-015), 마감 전 다음날 판단·야간 이벤트 시나리오(FR-017~022), 행동 기록·판단 갭(FR-023~025), 그리고 리스크 코칭 톤 원칙(FR-029). User Scenario 1~6을 화면 흐름으로 반영했다.
- User Flows: 플로우 1(개장 전 방어 브리핑), 플로우 2(포지션 등록·원칙 설정), 플로우 3(장중 알림·마감 전 판단), 플로우 4(행동 기록·인사이트), 플로우 5(이용 신청·승인 — 앞단 게이트). userflow의 화면 인벤토리 18개 화면과 ASCII 스케치를 화면 후보의 1차 근거로 삼았다.
- Additional Context: docs/idea.md(미반영 — PRD·userflow에 충분히 흡수됨), docs/DESIGN.md(반영 — Guard Blue 단일 액센트, 라이트/다크 결정 타일 교차 리듬, Pretendard 타이포, 결론 먼저·근거 아래 레이아웃, pill CTA, 단일 아티팩트 섀도, 리스크 코칭 보이스·금지 표현).

## Mockup Scope

- 18개 화면의 기본 상태 + 화면별 필요 상태(빈/로딩/오류/대기/승인/거절/선택 등).
- 결론 우선 표현, 행동/금지 라벨, 근거 점수 표시, 미준수 리스크 순서(`원인 → 취약 종목 → 손실 결과 → 확인할 지표`), 2차원 맵·수급·시나리오의 정보 구조를 더미 데이터로 시각화.
- 승인 게이트의 3가지 신청 상태(대기/승인/거절)와 관리자 검토 화면.
- 반응형은 모바일(단일 컬럼)·데스크톱(타일/그리드) 두 기준만 검토.

## Out of Scope

- 실제 인증/권한 로직(Supabase Auth 등), 관리자 시드·세션.
- DB 저장, API 연동, 실시간 시세·매크로·뉴스·실적 캘린더 데이터 소스.
- AI 종합 판단·9단계 장세 판정의 실제 산출 로직(목업은 더미 결론·점수로 표현).
- SMS·이메일 실제 발송, 인증번호 검증.
- 개인화 학습/파인튜닝 파이프라인, P2 기능의 백엔드.
- 세부 CSS 구현, 컴포넌트 내부 구조, 컴포넌트 추출 작업.

## Screen Flow

```text
[S001 이용 신청]
  -> [S002 승인 대기/상태]            (제출 후 / 거절·승인 결과 확인)
      -> [S004 환영/빠른 등록]        (승인 시 시작하기)
[S003 관리자 신청 관리]               (관리자 트랙 · S001 신청을 검토)
  -> [S002 승인 대기/상태]            (승인·거절이 신청자 상태에 반영)

[S004 환영/빠른 등록]
  -> [S005 아침 브리핑 대시보드]
      -> [S006 근거 상세]
      -> [S007 포지션 관리]
          -> [S008 위험선/알림 설정]
              -> [S009 매매 원칙]

[S010 장중 알림]                      (푸시/문자 진입)
  -> [S009 매매 원칙]                 (원칙 확인)
  -> [S006 근거 상세]                 (상세 보기)
  -> [S011 장중 포지션 변경]
      -> [S005 아침 브리핑 대시보드]  (최신 상태로 다시 판단)
[S012 장중 시황 요약]
  -> [S013 마감 전 판단]
      -> [S009 매매 원칙]

[S014 행동 기록 입력]
  -> [S015 판단 갭 리포트]
      -> [S017 개인화 인사이트]
      -> [S016 유사 상황 회상]
      -> [S018 오판 분석 리포트]
          -> [S017 개인화 인사이트]   (학습에 반영)
```

## [x] Screen S001 - 이용 신청 화면

**Purpose:** 이용 권한이 없는 방문자가 최소 정보로 이용을 신청하고 검토를 요청한다.

**User Moment:** 플로우 5 단계 1 — Scenario 5 "이 서비스, 일단 써볼 수 있나?"의 첫 접점.

**Primary Action:** `[신청하기]`

**Leads To:** S002(승인 대기/상태)

**States to Mock:** 기본(빈 폼), 입력 완료, 동의 미체크(제출 비활성), 제출 중(로딩), 동일 연락처 기존 신청 안내

**Review Focus:**
- [ ] 신청에 필요한 최소 입력(이름·연락처·동의)이 부담 없이 느껴지는가?
- [ ] 투자 경험·동기가 "검토 참고용"임이 분명한가?

**Done When:**
- [ ] 필수/선택 입력과 동의 체크가 구분되고, 제출 후 S002로 가는 흐름이 설명 가능하다.
