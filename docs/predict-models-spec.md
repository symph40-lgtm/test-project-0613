# 대가 방법론 예측 모델 시스템 기획서 (v1.4 — 체크포인트 판정 스케줄)

작성일: 2026-07-16 | 위치: `lib/predict/` (기존 M7 신호 시스템과 **완전 분리**)

## 0. 배경과 목적

기존 M7 판정(`lib/signal/`)은 사용자가 자체적으로 제시한 조건(분봉 상승/하락 RV1, 산·골 스윙 T6, FKS200 수급 T4·T5·T8 등)과 외부 기법이 섞여 있다. 사용자 지시(2026-07-16):

> "내가 제시한 조건들의 셋으로 판단하는 것과 대가들이 제시한 기법의 조건들로 판단하는 방법을 **각각 별도로 구현**하고, 실제 데이터와 비교해 보고자 해."

이 시스템은 **대가 5인의 방법론을 각각 독립 모델로** 구현하고, 과거 90거래일 실데이터로 정확도를 검증하며, 매일 장 마감 후 결과를 채점해 누적 정확도를 갱신하고, 그 정확도를 가중치로 하는 앙상블 최종 판정을 낸다.

- 기존 시스템과의 관계: 코드·테이블·문자 발송 모두 무관. 기존 판정에 어떤 영향도 주지 않는다.
- 사용자 자체 조건 모델("user" 모델)의 분리 구현은 **후속 단계** — 기존 `signal_judgments` 기록을 동일 라벨로 채점해 여섯 번째 모델로 편입 예정.

### 1.1 실제 추세 라벨 (그날 장 마감 후 확정)

"실제 그날의 주가 흐름"의 정량 정의 (`lib/predict/label.ts`, 파라미터는 config):

```
rOC   = (종가 - 시가) / 시가 × 100          # 당일 시가→종가 등락 (당일 매매 관점)
pos   = (종가 - 저가) / (고가 - 저가)        # 종가의 당일 레인지 내 위치

label = leverage  if rOC ≥ +1.2% AND pos ≥ 0.65     # 상승하고 고가권 마감
        inverse   if rOC ≤ -1.2% AND pos ≤ 0.35     # 하락하고 저가권 마감
        none      otherwise
```

- 시가 기준(전일 종가 아님): 아침 판정→당일 청산 매매의 실현 가능 구간이 시가~종가이므로.
- 종가 위치 조건: "크게 올랐다가 다 반납한 날"을 추세일로 오인하지 않기 위함.
- 백테스트가 라벨 분포를 함께 출력 — 극단적 불균형이면 임계 재조정.

### 1.2 채점과 학습 (매일)

- 장 마감 후(15:35 이후 아무 호출이나) 당일 라벨 확정 → 모델별 `correct` 채점 → 누적.
- 모델 가중치 = **리프트** (v1.1 개정 2026-07-16 — 90일 실측에서 균등가중이 피셔 단독보다 낮았던 희석 문제 해법):

```
평활 정확도 = (맞춘 수 + 1) / (판정 수 + 3)                 # 표본 적을 때 1/3로 수렴
우연 기준선 = Σ_v (모델의 v 판정 비율 × 실제 v 라벨 비율)      # 그 분포로 아무렇게나 질러도 맞을 확률
가중치     = max(평활 정확도 − 우연 기준선, 0)               # 우연 이하 모델은 0 = 침묵
```

- 전 모델 가중치 0(기록 없음)이면 균등 폴백. 백테스트 기록을 초기 이력으로 시딩 → 가동 첫날부터 실측 기반.
- **시각별 채점 (v1.6, 사용자 지정 2026-07-20)**: 14:00 확정은 하루 전체를 본 뒤라 정확도가 높은 게
  당연 — 실전 가치는 "빠른 판정의 정확도"다. 체크포인트별 판정은 매일 타임라인(revisions)째로
  채점·누적되고, 문자의 "이시각 실측적중"은 **그 시각 슬롯의 적중률**을 쓴다 (라이브 표본 20회
  미만이면 220일 백테스트 사전값 `config.checkpointPriors` — 09:00 44% · 09:30 49% · 10:30 58% ·
  14:00 62% 등).
- 채점·백필은 KIS 과거 분봉으로 언제든 재구성 가능 — 크론이 며칠 죽어도 다음 호출 때 소급 채점 (`/signal`의 "페이지 열려 있어야 축적" 제약이 없음).

### 1.3 앙상블 최종 판정

```
weight(m) = 리프트(m)                                  # 1.2의 리프트 가중치
score(v)  = Σ_{m: verdict(m)=v} weight(m) × confidence(m)
최종 판정  = argmax_v score(v)
강도(%)   = score(최종) / Σ_v score(v) × 100
```

출력: 각 모델의 판정·신뢰도 + 최종 판정 + 강도 %. 백테스트에서는 **워크포워드**(그날 이전 기록만으로 가중치 산출)로 재현해 미래 정보 누출 없이 평가.

## 2. 모델 사양 (대가별 독립 파일)

각 모델은 순수 함수 `(DayInput) → { verdict, confidence, reason }`. 모델 간 참조 금지.

### 2.1 crabel.ts — 토비 크레이블: 변동성 수축 + 시가 돌파 (ORB)

- Stretch = 직전 10일 `min(|시가-고가|, |시가-저가|)`의 평균 (크레이블 원전 정의)
- 수축 플래그: NR7 / (NR4 AND 인사이드바) — 전일 기준
- 09:00부터 1분봉 종가가 `시가 ± Stretch` 최초 돌파 → 그 방향 (상=leverage, 하=inverse)
- 돌파 후 10:30 전에 반대 레벨도 돌파(휩쏘) → `none`
- 신뢰도: 기본 0.55, NR7 +0.15, NR4&IB +0.10, 09:30 이전 돌파 +0.10 (상한 0.9). 무돌파 → `none` 0.5

### 2.2 raschke.ts — 린다 라쉬케: 추세일 증거 스코어

10:30 시점, 아침 레인지(09:00~10:30)에서 방향별 증거 수집:

- E1 갭 유지: |갭| ≥ 0.4×ATR14% 이고 갭 방향 유지(미채움) → 갭 방향 +1
- E2 레인지 확장: 현재가가 전일 고가 위(상) / 전일 저가 아래(하) → +1
- E3 시가-현재가 위치: 시가가 아침 레인지 하위 25% & 현재가 상위 75% → 상방 +1 (거울 대칭)
- E4 얕은 되돌림: 방향 극값 대비 되돌림 ≤ 아침 레인지의 40% → +1

한 방향 점수 ≥ 2 이고 반대 방향 < 2 → 그 방향. 신뢰도 `min(0.85, 0.45 + 0.15×점수)`.
그 외 `none` (아침 레인지 < 0.5×ATR이면 횡보 증거로 0.65, 아니면 0.5).
터틀 수프 등 역추세 셋업은 의도적으로 배제 (2026-07-16 검토 결론).

### 2.3 fisher.ts — 마크 피셔: ACD (A지점 확인·C지점 철회)

- OR = 09:00~09:15 (완성봉 15개) 고저
- 오프셋 = 0.15 × 직전 10일 평균 일중폭(고가-저가)
- A상 = OR고가 + 오프셋, A하 = OR저가 - 오프셋, C는 반대편 레벨
- **확인**: 1분봉 종가가 A 레벨 밖에서 **8분 연속**(OR의 절반) 유지 → 그 방향 판정
- **철회/반전**: A 확인 후 반대편 C 레벨 밖 5분 연속 → 판정을 반대로 전환 (10:30까지 마지막 상태가 최종)
- 신뢰도: 09:45 이전 A확인 0.8 / 이후 0.7 / C반전 0.6 / 무확인 `none` 0.5

### 2.4 dalton.ts — 스타이들마이어·달튼: 전일 가치영역 + 시가 유형

- 전일 가치영역(VA): 전일 전체 1분봉으로 가격대(500원 단위)별 거래량 프로파일 → POC 중심 70% 구간
- 시가 위치: VA 위 / 안 / 아래
- 시가 유형(09:00~09:30, ext-modules O1 규칙 재사용): open_drive(교차 0회·진행 ≥0.4%) / open_test_drive(교차 ≤2·역행 ≤0.3%) / open_auction(교차 ≥3) / undetermined
- 판정:
  - VA 밖 개장 + drive/test_drive가 VA 반대 방향 + 10:00에도 VA 밖 유지(수용) → 그 방향, 신뢰도 0.75 (open_drive면 +0.10)
  - VA 밖 개장했으나 10:30 전 VA 안으로 복귀(거부) → `none` 0.55 (80% 룰 역매매는 채택 안 함)
  - VA 안 개장 + auction → `none` 0.7 / + drive → 방향 0.55 / 그 외 `none` 0.5

### 2.5 grimes.ts — 애덤 그라임스: 레짐 + 풀백 재개 (일봉 중심)

- 레짐: 전일 종가 > SMA20 이고 SMA20 5일 기울기 > 0 → 상승 레짐 (거울 대칭)
- 풀백: 10일 최고가 이후 ≥ 2일 경과 AND 전일 종가가 SMA20의 ±1×ATR 이내 (평균 회귀 완료 상태)
- 트리거: 10:30 현재가가 전일 고가 돌파(상승 레짐) / 전일 저가 이탈(하락 레짐) → 그 방향 0.7
- 레짐·풀백 있고 트리거 없음, 또는 레짐 없음 → `none` 0.5
- 그라임스의 본령(통계 검증)은 모델이 아니라 **백테스트 하네스**에 구현: 무작위 기준선 대비 리프트, 워크포워드

본 피셔의 오프셋 0.15 → **0.25** 변형. 배경: ATR 11.8% 고변동 레짐(최근 60일)에서 본 피셔
방향적중 54%로 열화, 상수 스윕에서 문턱 상향(긴 OR·큰 오프셋)이 일관 개선 — 0.25가 최우수
(최근 60일 방향 65.8%). 단 in-sample 격자 탐색이라 즉시 채택하지 않고 **섀도 대조군**으로만
편입 (판정·문자 무관, 매일 채점). **승격 기준(사전 등록): 라이브 1개월에서 본 피셔 대비
방향적중 우위 지속 → 상수 교체 검토.** 시간 상수(OR 15분·확인 8분)는 유지 — 스윕상 효과가
오프셋과 중복이고 변형 수를 늘리면 다중 검정 오염.

## 3. 검증 프로토콜 (scripts/predict-backtest.ts)

1. 최근 90거래일(설정 가능) × 전체 1분봉 KIS 수집 → `.predict-cache/`에 캐시 (재실행 시 무통신)
2. 각 날짜의 라벨 확정 (1.1)
3. 각 모델에 그날 아침 데이터만 입력 → 판정 (미래 정보 차단: 일봉 이력은 전일까지, 분봉은 10:30까지)
4. 산출:
   - 모델별 3분류 정확도 / 방향 판정(`none` 제외) 시의 방향 적중률 / 커버리지(방향 판정 비율)
   - **무작위 기준선**: 모델의 판정 분포 × 라벨 분포 내적(우연 일치율) → 리프트 = 정확도 − 기준선
   - 앙상블: 워크포워드 정확도 + 최종 판정별 10:30→종가 평균 수익률 (경제적 가치)
5. `--seed` 플래그: 90일 기록을 `predict_model_days`/`predict_days`에 `source='backtest'`로 적재 (가동 초기 가중치)
