# 레버리지·인버스 신호 시스템 기획 설계서 (v2.4)

> **v2.4 변경**: R6·R7·R8 복구 (일일 손실 한도·분할 진입·해외 상품 특칙 유지). v2.3에서 조정한 R2 트레일링 기본값 -4%(설정 변경 가능)와 D2(나스닥100) 판정 제외·기록 전용 전환은 그대로 유지.

> **v2.2 변경 핵심**: 진입 판정을 "갭 조건" 기반에서 **2축 정렬 모델(Bias × Confirmation)**로 전환. 축 1 = 매크로·펀더멘털이 가리키는 방향과 강도(비중 연동), 축 2 = 시가 이후 실제 형성되는 방향 지속성. 갭은 필수 조건에서 제외하고 유형 분류·진입 방식 조절용으로 강등. 반전형(7/3형)·연속형(gap-and-go)·무갭형 세 경로 모두 수용.
>
> **v2.0**: "추세일 판별 엔진" 중심 재설계. 매일 장 초반에 ①추세일/횡보일 판별 → ②방향 판정 → ③방향에 맞는 상품(레버리지/인버스) 진입의 3단계 구조. FKS200을 1차 신호원, 타국 지수 정합성을 2차 필터로 사용.

작성일: 2026-07-04 | 대상: 삼성전자·SK하이닉스 레버리지/인버스 ETP 단기 트레이딩 자동화

## 1. 목적과 범위

### 1.1 목적
사용자의 실전 검증된 트레이딩 규칙(2026년 6월 NFP 역발상 매수 성공사례 및 축적된 매매 원칙)을 정형화된 신호 체계로 변환하고, 조건 충족 시 알림·주문 지침을 제공하는 시스템을 구축한다.

### 1.2 범위
- 대상 상품: 국내 상장 삼전/하닉 관련 레버리지·인버스 ETF/ETN, 홍콩 CSOP 2배, 영국 3배 ETP(HNX3 등)
- 기능 범위: 데이터 수집 → 신호 판정(스코어링) → 알림 → (선택) 반자동 주문 보조 → 트레일링 스탑 관리 → 매매일지 자동 기록
- 1단계는 **알림 시스템**으로 시작하고, 자동 주문은 후순위(6장 로드맵 참조)

### 1.3 설계 원칙
1. **사람이 최종 결정, 시스템은 판단 보조**: 성공사례의 핵심이 "장중 분위기 해석"이었으므로, 완전 자동화보다 체크리스트 점수화 + 알림이 목적에 부합
2. **손실 제한 우선**: 모든 진입은 시세포착주문(-3%) 동반을 전제로 설계
3. **예외 규칙의 명시화**: "폭락 직후 갭상승엔 인버스 금지" 같은 예외를 하드코딩

## 2. 규칙 정형화 (Rule Engine 명세)

사용자 문서의 조건을 기계 판독 가능한 신호로 변환. 각 신호는 [필수/가점] 구분과 데이터 소스를 가진다.

|---|---|---|---|
| C1 | 이벤트 캘린더 | 당일/익일 NFP·CPI·FOMC·실적발표·선물옵션만기 여부 태깅 | 경제캘린더 API |
| C2 | 리밸런싱 월 | 분기 첫째 달(1·4·7·10월)=순풍, 2·5·8·11월(MSCI)·분기말=역풍 | 달력 |

| C4 | 미 금리 | 2년물·10년물 전일 변동, 상승 지속/안정/하락 3분류 | 시세 API |
| C5 | 밤사이 미국장 | 나스닥·필라델피아반도체(SOX) 등락률 | 시세 API |

### 2.2 진입 원리 (v2.2 — 2축 정렬 모델)

**갭은 필수 조건이 아니다.** 레버리지 진입 경로는 세 가지 모두 유효하다: ①갭하락 출발 → 반전 후 상승 지속(7/3형), ②갭상승 출발 → 계속 상승(gap-and-go), ③무갭 출발 → 계속 상승. 인버스도 거울상으로 세 경로(갭상승→반전 하락, 갭하락→연속 하락, 무갭→연속 하락). 갭의 역할은 진입 가부 결정이 아니라 **유형 분류(T7)와 진입 방식 조절**(큰 갭일수록 시초 진입 금지, 확인 후 진입)로 한정한다.

진입은 두 축의 정렬로 판정한다:

**축 1 — Bias (매크로·펀더멘털 방향 점수)**: 오늘 가격이 "가야 마땅한" 방향과 그 강도. 구성: 금리·환율 방향(C3·C4), 전일 이벤트·지표의 서프라이즈 방향(L10), 이익 컨센서스 유지 여부(L8), 낙폭/상승폭의 원인 성격(L7). 산출: 상방 Bias / 하방 Bias / 중립, 각각 강도 1~3등급. **매크로 개선의 강도가 클수록 허용 비중이 커진다** — 이벤트가 좋아지고 펀더멘털이 유지되면, 그 강함의 정도에 비례해 레버리지 사용.

**축 2 — Confirmation (가격 행동 확인)**: 시가 이후 실제로 형성되는 방향 지속성. 구성: FKS200의 방향 형성과 유지(반전이든 연속이든), 실시간 DC1/DC2 (2.5.6), 되돌림 깊이(T3), 방향 전환 횟수(T6). 판정: "한 방향이 형성되어 어느 정도 유지되는 구간이 확인되는가."

**진입 규칙**: Bias 방향 = Confirmation 방향으로 정렬될 때만 진입. 정렬의 대표 시나리오:
| 시나리오 | Bias | 가격 초반 | 판정 |
|---|---|---|---|
| 괴리 후 반전 (7/3형, 성공사례 원형) | 상방(매크로 개선 + 펀더멘털 무손상) | 하락 지속 = **괴리(이상 신호), 감시 강화** → 반전 발생 → 반전 후 진행 강도 확인 | 강한 진행 확인 시 레버리지. 괴리 자체는 진입 신호가 아니라 "반전을 기다리는 상태" |
| 순방향 연속 (gap-and-go) | 상방 | 처음부터 상승 지속 (갭 유무 무관) | Confirmation 충족 즉시 레버리지, 비중은 Bias 강도 비례 |
| 하방 정렬 | 하방(매크로 악화·악재) | 하락 형성·지속 (갭 유무 무관) | 인버스 |
| 불일치 | 상방 | 하락 지속·반전 기미 없음 | **진입 금지, 관찰만** — Bias가 맞다고 가격에 역행해 미리 사지 않는다. 가격이 돌아설 때까지 대기 |

### 2.2.1 레버리지(롱) 진입 신호

**필수 조건 (모두 충족해야 후보)**
| 코드 | 조건 | 판정 기준 |
|---|---|---|
| L1 | Bias 상방 | 축 1 판정이 상방 (금리·환율 안정 이상 + 이벤트·지표 우호 방향), 강도 1~3 산출 |
| L2 | 금리·환율 안정 | C3 하락 or 보합 AND C4 상승 아님 (Bias의 하위 구성이자 단독 필수) |
| L3 | 방향 형성 확인 | FKS200이 상방으로 방향을 형성·유지: (a)하락 → 축소 → 상승 반전 후 **반전 방향 진행이 강하게 유지**(되돌림 얕음, 실시간 DC1 상방 우위), 또는 (b)초반부터 상승 지속. 반전형은 반전 직후가 아니라 **반전 후 지속 확인 뒤** 진입 |
| L4 | 진입 시간대 | 09:30 ~ 10:30 사이에만 진입 (09:00~09:30은 관찰 전용) |
| L5 | 외인 수급 (3요소) | **순매수 전환을 요구하지 않음** (구조적 매도 국면 대응). ①수준: 10:00까지 누적 순매도 ≤ 기준치(초기값 1.1조) ②기울기: 30분 구간별 순매도 증가분이 감속 중(뒤 구간 ≤ 앞 구간) ③상대강도: 당일 매도 페이스가 최근 20일 일평균 순매도의 1.2배 이하. 3개 중 ①+② 필수, ③은 가점 |

| 코드 | 조건 |
|---|---|
| L6 | 직전 1~3일 누적 -12% 이상 과대 낙폭 (V반등 후보) |
| L7 | 낙폭의 원인이 수급·지정학·소송 등 비(非)실적 요인 |

**금지 조건 (하나라도 해당 시 진입 불가)**
- X1(수정): 갭상승 +2% 초과 출발일은 **시초 직후 추격 진입만 금지** — L3(b) 상승 지속이 09:30 이후 확인되면 진입 허용 (gap-and-go 유형 수용. 단 큰 갭일수록 확인 구간을 길게)
- X2: 당일 밤 바이너리 이벤트(NFP/FOMC/실적) 예정인데 오버나잇 계획인 경우 — 단, 예외규칙 E1 참조
- X3: 당일 손실 한도 이미 도달

**필수 조건**
| 코드 | 조건 | 판정 기준 |

**가점 조건**
- S5: 나스닥·일본 등 타국 하락 중인데 한국(하닉)만 상승 — 디커플링 과열
- S6: 악재 뉴스 발생(공급과잉 우려, 소송, 규제 등)
- S7: 매크로 발표가 컨센서스 수준 → 금리인상 가능성 유지

**금지 조건 (2026.6월 데이터 검증에서 발견된 핵심 예외)**
- **XS1: 직전 1~3일 폭락(누적 -12% 이상) 후의 갭상승은 V반등이므로 인버스 절대 금지** (6/9 갭+7.3%→장중 +8.1%, 6/25 갭+11%→장중 +1.8% 사례)

## 2.5 추세일 판별 엔진 (v2.0 신규 — 시스템의 중심 모듈)

### 2.5.1 문제 정의
매일의 장은 크게 세 가지다: ① 초반에 방향을 잡고 종일 유지하는 **추세일**(갭하락 후 계속 하락 → 인버스 / 갭상승 후 계속 상승 → 레버리지), ② 방향 없이 왔다갔다 하는 **횡보일**(레버리지 상품엔 최악 — 스탑 왕복 손실), ③ 초반 방향이 장중 뒤집히는 **반전일**(V반등·페이드, 기존 2.2~2.3의 셋업). 시스템의 1차 임무는 09:00~10:00 데이터로 오늘이 어느 유형인지 조기 판별하는 것이다.

사용자 관찰의 정형화: "FKS200은 한번 방향을 잡으면 그 방향이 유지되는 경향이 있고, 국내 현물은 변동이 심해 방향을 못 잡는다" → 코스피200 선물은 외국인·프로그램 매매가 주도해 방향성이 깨끗하고, 현물 개별주는 노이즈가 크다. 따라서 **방향 판정은 선물(FKS200)로 하고, 실제 매매는 하닉/삼전 레버리지·인버스로 실행**하는 이원 구조가 합리적이다.

### 2.5.2 추세일 판별 신호 (T-시그널)

| 코드 | 신호 | 판정 기준 | 논리 |
|---|---|---|---|
| T1 | 시초 레인지 이탈 | 09:00~09:30 선물 고가·저가(Opening Range)를 09:30 이후 한 방향으로 이탈 후 재진입 없음 | 추세일의 고전적 정의(ORB). 레인지를 위로 깨면 상승 추세 후보, 아래로 깨면 하락 추세 후보 |
| T2 | VWAP 편측성 | 선물 가격이 당일 VWAP의 한쪽에만 머무는 시간 비율 ≥ 80% (10시 기준) | 추세일은 VWAP 재터치 없이 이격을 벌림. 횡보일은 VWAP을 반복 관통 |
| T3 | 되돌림 깊이 | 직전 스윙 파동의 되돌림이 40% 미만으로 얕음 | 얕은 눌림 = 매수(매도)세가 기다리지 않고 쫓아옴 = 추세 지속 신호 |
| T4 | 외국인 선물 수급 | 외국인 코스피200 선물 순매수(도) 잠정치가 한 방향으로 누적 증가. 현물이 구조적 매도 국면일 때는 부호가 아닌 **기울기 변화(매도 감속=상방, 매도 가속=하방)**로 판정 | 추세일의 동력원. 외인 선물 방향과 가격 방향 일치 시 신뢰도 급상승 |
| T5 | 프로그램 매매 방향 | 차익+비차익 프로그램 순매수(도)가 가격과 동일 방향 | 선물 주도 장세의 현물 전이 확인 |
| T6 | 방향 전환 횟수 | 09:00~10:00 사이 5분봉 기준 고점/저점 갱신 방향의 전환 횟수 ≤ 2회 | 3회 이상 뒤집히면 횡보일로 분류, 당일 추세 매매 금지 |
| T7 | 갭과 초반 방향의 일치 | 갭 방향과 첫 30분 진행 방향이 같음 (gap-and-go) | 갭+동방향 진행 = 추세일 확률 상승. 갭과 역방향 진행 = 반전일 후보(2.2/2.3 셋업으로 라우팅) |
| T8 | 거래대금 확장 | 진행 방향 5분봉의 거래대금이 반대 방향 봉 대비 우위 | 방향에 돈이 실리는지 확인 |

### 2.5.3 정합성 검사 (Cross-Market Divergence — 2차 필터)

사용자 관찰의 정형화: "일본은 오르는데 한국만 내리면 이상하다" → 동조화 이탈은 두 가지 의미를 가진다.

| 코드 | 검사 | 해석 |
|---|---|---|
| D1 | 니케이225 선물 vs FKS200 | 같은 방향 = 추세 신뢰도 가점. 역방향 = 이탈 경보 |
| D2 | 나스닥100 선물(장중 시간외) vs FKS200 | **판정 미사용, 기록만** (v2.3): 실측상 나스닥 하락일에도 한국 반도체 상승 사례 확인 — 외인 리밸런싱·국내 고유 동력 국면에서 정합성 낮음. 학습 데이터로만 축적해 유효 레짐 여부를 8장에서 사후 검증 |
| D3 | 대만 자취안(반도체 동조) vs 하닉 | 반도체 섹터 고유 흐름 분리 |
| D4 | 이탈 원인 분류 | 한국 고유 악재·호재 존재 시(뉴스·수급) → 이탈은 정당, 추세 지속 가능. 고유 원인 부재 시 → 과매도·과열 왜곡, **반전(평균회귀) 셋업으로 라우팅** (성공사례: 일본 상승 속 한국만 하락 → 과대 눌림 판단 → 롱) |

핵심 규칙: **정합(타국과 동방향) 추세는 추세추종으로, 이탈(한국만 역방향) 상태는 원인이 없을 때만 역발상으로.** 이탈인데 한국 고유 원인이 있으면(예: 6/23 전쟁+수급) 그 방향의 추세일로 취급한다.

### 2.5.6 방향 지속률(Directional Consistency) — 추세일의 정량 라벨 (v2.1)

추세일을 사람의 인상이 아니라 수치로 정의한다. 두 지표를 병행 산출:

| 지표 | 정의 | 해석 |
|---|---|---|
| DC1 (동방향 봉 비율) | 당일 **10분봉** 중, 그날의 순방향(종가-시가 부호)과 같은 방향으로 마감한 봉의 비율 | **60% 이상 = 추세일 후보** (실측 근거: 7/3 +30% 수익일의 10분봉 양봉 비율이 65% — 강한 추세일도 봉 단위로는 눌림봉이 3분의 1 섞임), 50% 부근 = 횡보일 |
| DC2 (효율비, Efficiency Ratio) | \|종가-시가\| ÷ Σ\|10분봉 변화폭\| (0~1) | 이동 경로 대비 순이동 거리. DC1이 60%대여도 양봉의 평균 크기가 음봉을 압도하면 DC2가 높게 나옴 — 봉 개수가 못 잡는 '크기의 비대칭'을 포착 |

**운용 원칙**
- 임계값 θ(초기값: **10분봉 DC1 60%** / ER 0.30)는 **고정 상수가 아니라 최적화 대상 파라미터** — 8장 학습 시스템에서 어떤 수치가 손익을 극대화하는지 탐색 (그리드: 55/60/65/70%, 봉 주기 5분/10분/15분도 함께 탐색)
  - *(2026-07-05 운용 변경: 봉 주기 5분 · DC1 55% · ER 0.25로 시작 — 판정 확정 최소 대기를 30분→15분으로 줄이기 위한 사용자 결정. 잘은 봉의 노이즈 증가에 맞춰 임계값 비례 보정. 실데이터 축적 후 Stage 3에서 재탐색)*
- DC1 단독 판정의 허점 보완: 60%는 횡보일과의 경계가 가까우므로, **추세일 확정은 DC1 ≥ θ AND DC2 ≥ 기준의 동시 충족**으로 판정 (봉 개수 + 크기 비대칭 이중 확인)
- 두 지표는 사후 라벨(그날이 추세일이었는가)로 쓰이며, 장중에는 "현재까지의 DC1/DC2"를 실시간 계산해 T-스코어 보조 및 **추세 붕괴 청산 신호**(보유 중 DC1이 임계 밑으로 하락 = 추세 소멸 → 조기 청산)로도 사용
- 상방 지속일 = 레버리지 적합일, 하방 지속일 = 인버스 적합일, 중간 = 매매 부적합일 — 3클래스 라벨

### 2.5.7 6월 데이터 재검증 (추세일 관점)

## 4. 스코어링 체계 (판정 로직)

### 4.1 레버리지(롱) 스코어
- 필수 L1~L5: 하나라도 미충족 → 판정 "대기"
- 가점: L6(+3), L7(+2), L8(+2), L9(+1), L10(+2), L11(+1) — 만점 11
- 판정: 필수 충족 + 가점 5점 이상 = "진입 후보(1/3 비중)", 8점 이상 = "강한 신호(분할 계획 전체 가동)"
- 금지 X1~X3 해당 시 점수 무관 차단

### 4.2 인버스 스코어
- 필수 S1~S4 충족 + 가점 S5(+2), S6(+2), S7(+1) — 만점 5
- 판정: 필수 충족 + 2점 이상 = 진입 후보. **XS1·XS2는 최우선 차단**
- 인버스는 기본 비중을 롱의 절반으로 (R5)

### 4.3 추세일(T) 스코어 — 최상위 판정 (v2.0)
- T1(+3), T2(+2), T3(+1), T4(+3), T5(+1), T7(+2), T8(+1) — 만점 13 / T6 위반 시 즉시 "횡보일" 선언
- 판정: **8점 이상 + D1·D3 정합** = 추세일 확정 → 방향대로 추세추종 진입 (UP=레버리지, DOWN=인버스)
- 5~7점 = 약한 추세 → 1/3 비중만, 트레일링 타이트(-2%)
- 4점 이하 or T6 위반 = 횡보일 → 당일 추세 매매 금지, 반전 셋업(2.2~2.3)만 감시
- 크로스마켓 이탈(D4) 시: 한국 고유 원인 있음 → 추세로 취급 / 원인 없음 → 역발상 셋업으로 라우팅
- 우선순위: 예외 필터(X·XS) > 폭락 후 V반등 분기 > T 스코어 > L/S 셋업 스코어

### 4.4 판정 예시 (2026.6월 실데이터 기반 백테스트 시나리오)
| 날짜 | 상황 | 시스템 판정 | 실제 결과 |
|---|---|---|---|
| 6/9 | 이틀 -17% 후 갭+7.3% | 롱: L6·L7 충족이나 X1(갭+2%초과)로 시초 진입 차단 → 장중 눌림 시 재판정 / 인버스: XS1 차단 | +15.9% (인버스 진입했다면 대참사) |

## 6. 개발 로드맵

**Phase 1 — 장전 브리핑 + 추세일 판별 알림 봇 (3~4주)**
- Python + 텔레그램 봇. 08:30 브리핑(당일 모드 판정), 09:00~10:30 1분 주기 T-스코어 산출
- 산출물: "추세일/횡보일/반전일 + 방향 + 스코어" 알림, 시세포착주문 수치 계산
- 성공 기준: 6월 급등/급락일 재현 테스트에서 4.4 표와 동일 판정. 특히 횡보일을 횡보일로 걸러내는 특이도(진입 안 함 판정의 정확도)를 승률보다 우선 검증 — 추세일 오탐 1회가 정탐 2회의 이익을 상쇄하기 때문
- 주의: T-스코어의 분봉 단위 검증에는 1분/5분봉 과거 데이터가 필요. KIS API 분봉 조회 또는 수동 축적으로 확보

**Phase 2 — 페이퍼 트레이딩 + 백테스트 (4주)**
- 알림대로 가상 매매 기록 1개월 → 패턴별 승률·손익비 측정
- 6개월 과거 데이터 백테스트 (일별 OHLC + 수급 데이터 확보 필요: KRX 정보데이터시스템에서 CSV 다운로드)
- 임계값 튜닝: 갭 기준(±2%), 과대낙폭 기준(-12%), 외인 매도 기준(1.1조)의 민감도 분석

**Phase 3 — 반자동 주문 + 트레일링 관리 (선택)**
- 알림 내 승인 버튼 → API 주문 실행, 포지션 보유 중 R2 트레일링 수정 알림 자동화
- 전제: Phase 2에서 실계좌 대비 신호 정확도 검증 완료

**Phase 4 — 지속 학습 루프 가동 (8장, Phase 1과 병행 시작 가능)**
- 장후 배치(라벨링·예측 대조·원인 주석 수집)는 Phase 1부터 즉시 가동 — 데이터 축적이 병목이므로 가장 먼저 시작할 것
- Stage 1 규칙 발견 리포트는 2개월 축적 후, Stage 2 확률 모델은 6개월+ 후 착수
- 버전 관리: 모델·파라미터·규칙 변경 이력을 매매일지와 연결해 "어떤 버전이 어떤 성과를 냈는지" 추적 가능하게 설계

## 7. 리스크 및 유의사항

1. **표본 한계**: 검증 데이터는 28거래일(초고변동 구간)로, 평온장에서는 신호 발생 빈도가 월 3~5회로 감소하고 V반등 패턴 승률이 하락할 수 있음. Phase 2 백테스트로 보정 필수
2. **과최적화 경계**: "1.1조", "-12%" 같은 임계값은 특정 구간에 맞춘 수치. 범위 테스트(0.8~1.4조, -8~-15%)로 강건성 확인
3. **6/23형 돌발 리스크**: 평범한 갭으로 출발해 장중 -12% 가는 날은 어떤 진입 규칙도 못 피함 → R6(일일 한도)와 R1(즉시 스탑)이 유일한 방어선
4. **레버리지 ETP 구조 비용**: 당일 청산 원칙(R3)이 변동성 끌림을 차단하는 핵심 장치이므로, 예외(R4)는 반드시 소량 원칙 유지
5. 본 문서는 판단 보조 시스템 설계이며 투자 결과에 대한 책임은 사용자에게 있음. 자동매매 관련 규정 및 증권사 약관 준수 필요

## 8. 지속 학습 시스템 설계 (v2.1 신규)

목표: "추세일(DC1 ≥ θ)이었던 날의 사전 조건을 학습해, 다음 추세일을 장 초반에 더 정확히 예측"하고, 새 데이터가 쌓일 때마다 시스템이 스스로 개선되는 구조.

### 8.1 라벨링 파이프라인 (과거 데이터 분석)

**피처 (그날 아침에 알 수 있었던 것 — 예측 입력)**

### 8.2 학습 단계 (데이터 양에 맞는 점진 고도화)

**Stage 1 — 규칙 발견 (표본 ~60일)**: 조건부 통계표. "환율 하락 + 전일 3일 누적 -10% 이하 → 상방추세일 확률 X%" 식의 단순 크로스탭. 해석 가능하고 과적합 위험 낮음. 산출물은 사람이 읽는 월간 패턴 리포트
**Stage 2 — 확률 모델 (표본 120일+)**: 로지스틱 회귀 or 얕은 그래디언트 부스팅으로 3클래스 확률 P(상방추세/하방추세/비추세) 예측. 08:30 버전(장전 피처만)과 09:30 버전(장초반 피처 포함) 두 모델 운용
**Stage 3 — 임계값·파라미터 최적화**: θ(10분봉 DC1 60%, 봉 주기 포함), 갭 기준(±2%), 과대낙폭(-12%), T-스코어 컷 등을 그리드 탐색. **최적화 목표는 적중률이 아니라 "거래비용 차감 후 순손익 × 최대낙폭 페널티"** — 적중률 최적화는 작은 이익 다수 + 큰 손실 소수 함정에 빠짐

### 8.3 지속 학습 루프 (운영 사이클)

```
[매일 15:40 장후 배치]
  1. 당일 5분봉 수집 → DC1/DC2 계산 → 라벨 확정
  2. 아침 예측값 vs 실제 라벨 대조 → 예측 로그 기록
  3. 피처 스냅샷 저장, 사용자에게 "원인 주석 1줄" 입력 요청 (텔레그램)
  4. 추세일이었는데 예측 실패한 날 → '오답 노트'에 자동 등재, 주말 리뷰 대상

[매월 1회]
  5. 워크포워드 재학습: 직전 N개월로 학습 → 다음 달 검증 (미래 데이터 누출 금지)
  6. 챔피언/챌린저: 신규 모델은 1개월 페이퍼 예측으로 기존 모델과 병행 비교 후 성능 우위 시에만 교체
  7. 파라미터 재탐색 (θ 등) 및 민감도 리포트

[분기 1회 or 레짐 변화 감지 시]
  8. 드리프트 점검: 최근 승률·손익비가 학습기 대비 통계적으로 악화되면 경보
     (예: 환율 레짐 전환, 외인 매도 국면 종료, 금리 사이클 전환 시 과거 패턴 무효화 가능)
  9. 무효화된 규칙은 삭제가 아니라 '레짐 조건부'로 보존 (예: "고환율 레짐에서만 유효")
```

### 8.4 학습 시스템의 한계와 안전장치
- **표본 부족이 최대 적**: 추세일은 한 달에 3~9일. 1년 모아야 표본 ~60개. Stage 2 이전에 복잡한 모델(딥러닝 등) 금지
- **예측 확신도와 비중 연동**: P(추세) 60%대 = 1/3 비중, 80%+ = 정상 비중. 모델이 틀려도 살아남는 구조가 모델 성능보다 중요
- 학습 결과가 어떤 경우에도 하드 블록(X·XS 예외 필터)과 리스크 규칙(R1~R8)을 무효화할 수 없음 — 학습은 "언제 진입할까"만 개선하고, "얼마나 잃을 수 있나"는 규칙이 고정
