# 확장 신호 모듈 상세 기획서 (v1.0)

작성일: 2026-07-05 | 상위 문서: 「레버리지·인버스 신호 시스템 기획 설계서 v2.4」의 확장 모듈 스펙
목적: 검증된 외부 기법 7종을 기존 시스템(Bias × Confirmation 2축, T-스코어, R-리스크 규칙)에 편입하기 위한 구현 수준 명세

---

## 0. 공통 설계 원칙

1. **모든 확장 신호는 독립 on/off 플래그를 가진다** — 설정 파일에서 개별 활성화. 기본값은 전부 OFF로 배포하고, 백테스트 통과분만 ON
2. **기존 필수 조건(L·S)과 하드 블록(X·XS)을 대체하지 않는다** — 확장 신호는 가점/보조 확인/파라미터 옵션으로만 작동
3. **모든 신호값은 daily_features 테이블에 기록** — 판정에 안 쓰더라도 축적해 학습 시스템(상위 문서 8장)의 피처로 자동 편입
4. 신호 코드 체계: N(수축), O(시가유형), B(베이시스), W(시장폭), V(변동성지수), A(ATR스탑), C(마감증폭)

## 1. N1 — NR7 전일 레인지 수축 (추세일 선행 예고)

### 1.1 정의와 논리
전일의 일중 레인지(고가-저가)가 직전 7거래일 중 최소이면 NR7. 변동성은 수축과 확장을 반복하므로, 수축 극단 다음 날은 확장(=추세일) 확률이 높아진다. **방향은 알려주지 않는다** — 방향 판정은 기존 Bias/Confirmation의 몫이고, N1은 "내일 추세일 가능성 상향" 예고만 담당.

### 1.2 계산
```
range(d) = high(d) - low(d)
NR7(d) = range(d) == min(range(d-6) ... range(d))
NR4(d) = range(d) == min(range(d-3) ... range(d))          # 완화 버전
IB(d)  = high(d) < high(d-1) AND low(d) > low(d-1)          # inside bar, 결합 시 신뢰도 상승
signal = NR7 or (NR4 and IB)                                 # 초기 규칙
```
- 대상: 하닉 본주 일봉 + 코스피200 선물 일봉 각각 계산 (둘 다 True면 강신호)

### 1.3 데이터
| 항목 | 소스 | 주기 |
|---|---|---|
| 하닉 일봉 OHLC | KIS API 일봉 조회 | 장후 1회 |
| 코스피200 선물 일봉 | KIS API | 장후 1회 |

### 1.4 통합 지점
- 장전 브리핑(08:30)에 "N1: 변동성 수축 상태 — 금일 추세일 확률 상향" 플래그 출력
- T-스코어 +1 가점 (본주·선물 동시 True면 +2)

### 1.5 파라미터·백테스트
- 튜닝: 룩백 4/7/10일, 본주 단독 vs 선물 단독 vs 동시
- 검증: NR7 다음 날의 DC1 분포 vs 전체 평균 비교 (추세일 비율 리프트가 유의한지). 표본 최소 6개월

### 1.6 주의
- 연속 NR7(수축이 수일 지속)은 확장이 미뤄지는 중 — 플래그는 유지하되 가점 중복 금지
- 이벤트 대기(FOMC 전날 등)로 인한 수축은 자연 수축과 성격이 다름 → C1 이벤트 플래그와 동시 True면 "이벤트 해소 방향으로 확장" 주석 추가

## 2. O1 — 시가 유형 분류 (Open Type, Market Profile 계열)

### 2.1 정의와 논리
09:00~09:30 가격 행동을 3유형으로 분류해 그날의 성격을 조기 판정한다. 짐 달튼의 Market Profile 시가 분류를 1분봉으로 근사 구현.

| 유형 | 정의 | 함의 |
|---|---|---|
| open_drive | 시가에서 한 방향으로 즉시 진행, 시가 재터치 없음 | 추세일 확률 최고. 기관 주도 확신 매매 |
| open_test_drive | 시가 반대편을 짧게 테스트 후 방향 전환·유지 | 추세일 확률 높음. 7/3형(초반 눌림 후 반전)과 동형 |
| open_auction | 시가를 중심으로 양방향 반복 교차 | 횡보일 확률 높음. 추세 매매 부적합 |

### 2.2 계산 (1분봉, 09:00~09:30 구간)
```
open_px = 09:00 시가 (선물 기준)
cross_count = 가격이 open_px를 상→하 또는 하→상으로 관통한 횟수
max_adverse = 최종 방향과 반대쪽으로 open_px 대비 최대 이탈폭 (%)
final_dir  = 09:30 가격 vs open_px 부호

if cross_count == 0 and |09:30가 - open_px| ≥ drive_min(초기 0.4%):
    open_type = open_drive
elif cross_count ≤ 2 and max_adverse ≤ test_max(초기 0.3%):
    open_type = open_test_drive
elif cross_count ≥ 3:
    open_type = open_auction
else:
    open_type = undetermined   # 판정 보류, 10:00 재판정
```

### 2.3 데이터
- 코스피200 선물 1분봉 실시간 (KIS API 체결/분봉). 09:30, 10:00 두 시점 판정

### 2.4 통합 지점
- open_drive → T-스코어 +3, Confirmation 축의 "방향 형성" 조기 충족 인정 (L3/S3 판정 가속)
- open_test_drive → T-스코어 +2, 반전형 시나리오(2.2 괴리 후 반전)와 결합 시 진입 신뢰도 상향
- open_auction → T6(방향 전환 횟수)와 결합해 횡보일 조기 선언, 당일 추세 매매 금지 권고
- undetermined → 판정 유보, 신호 없음

### 2.5 파라미터·백테스트
- 튜닝: drive_min 0.3~0.6%, test_max 0.2~0.5%, cross 임계 2~4회, 판정 창 30분/60분
- 검증: 유형별 당일 DC1 분포·장중 수익률 분포 비교. open_drive의 추세일 적중률이 기준선 대비 유의하게 높은지

### 2.6 주의
- 대형 갭(±3% 이상) 출발일은 시가 자체가 극단이라 분류 왜곡 → 갭 크기별로 분리 집계
- 09:00~09:05 호가 불안정 구간은 cross_count 계산에서 제외 옵션 검토

## 3. B1 — 선물 베이시스 (프로그램 매매 선행 신호)

### 3.1 정의와 논리
베이시스 = 코스피200 선물가 - 코스피200 현물지수. 베이시스가 평소 대비 확대(콘탱고 심화)되면 차익거래 프로그램 매수가 기계적으로 유입되고, 축소·역전(백워데이션)이면 매도 압력. 즉 B1은 T5(프로그램 매매 방향)의 **선행 지표**다.

### 3.2 계산
이론가 계산(금리·배당 추정 필요)은 오차원이 많으므로 **통계적 정규화**로 구현:
```
basis(t) = futures_px(t) - kospi200_index(t)
basis_z(t) = (basis(t) - mean_20d(basis, 동시간대)) / std_20d(basis, 동시간대)
basis_slope(t) = basis(t) - basis(t-30min)                 # 장중 변화 방향

신호:
  basis_z ≥ +1.0 AND basis_slope > 0  → 프로그램 매수 유입 예상 (상방 가점)
  basis_z ≤ -1.0 AND basis_slope < 0  → 프로그램 매도 압력 (하방 가점)
```
- "동시간대" 비교가 핵심: 베이시스는 장중 시간대별 패턴이 있으므로 같은 시각끼리 z-score 산출

### 3.3 데이터
- KIS API: 선물 현재가 + 코스피200 지수, 1분 주기. 과거 20일 장중 베이시스 시계열 축적 필요 (가동 후 20일간은 수집만)

### 3.4 통합 지점
- T-스코어: 방향 일치 시 +1 (T5와 중복 가점 방지 — B1과 T5가 동시 충족이면 합산 최대 +2로 캡)

### 3.5 주의 (판정 제외 조건)
- **만기 주간(선물옵션 만기 D-3 이내): B1 판정 전면 제외** — 만기 수렴으로 베이시스가 신호가 아닌 노이즈가 됨
- 배당락 시즌(12월, 3~4월 배당 기준일 전후)은 베이시스 레벨이 구조적으로 이동 → z-score 룩백을 10일로 단축
- 급변동 순간의 현선물 갱신 시차(수 초)로 스파이크 발생 → 3분 이동평균으로 평활 후 판정

## 4. W1 — 시장 폭 (Breadth, 진성 추세 검증)

### 4.1 정의와 논리
진짜 추세일은 지수만이 아니라 다수 종목이 같은 방향으로 움직인다. 하닉 한 종목이 지수를 끌어올리는 왜곡 장세를 구분하는 것이 목적. 단, 왜곡 장세라도 매매 대상이 하닉 자체이므로 **차단이 아니라 태그** — 지수 추종 신호(FKS200 기반 T들)의 신뢰도만 조정한다.

### 4.2 계산 (10:00 시점, 이후 1시간마다 갱신)
```
breadth = 코스피 상승종목수 / (상승종목수 + 하락종목수)
divergence = kospi200_등락률 - kospi_전체등락률(동일가중 근사 또는 코스닥 포함 확장)

판정:
  breadth ≥ 0.70 AND 지수 상승  → 진성 상방 추세 태그 (T-스코어 +1)
  breadth ≤ 0.30 AND 지수 하락  → 진성 하방 추세 태그 (+1)
  0.45 ≤ breadth ≤ 0.55 AND |지수 등락| ≥ 1%  → 대형주 왜곡 태그
      → FKS200 기반 T-신호 가점을 절반으로 감액 (지수 추세가 하닉 단독 현상일 가능성)
      → 단, 하닉 개별 DC1이 강하면 개별주 신호로는 유효 유지
```

### 4.3 데이터
- KIS API 등락 종목수 집계 또는 KRX 정보데이터시스템. 10분 주기면 충분

### 4.4 백테스트
- 진성 태그일 vs 왜곡 태그일의 "지수 추세 → 하닉 추세 전이율" 비교. 왜곡일에 지수 신호를 감액하는 것이 손익 개선에 기여했는지 검증

## 5. V1 — VKOSPI 피크아웃 (V반등 진입 보조 확인)

### 5.1 정의와 논리
급락일에 변동성지수(VKOSPI)는 공포와 함께 급등하다가, 매도 절정이 지나면 가격보다 먼저 꺾이는 경우가 있다. V반등 셋업(상위 문서 L6·L7)에서 "반전이 진짜인지"를 보조 확인한다.

### 5.2 계산
```
전제 조건: 당일 하닉/지수 급락 중 (장중 -3% 이상) AND V반등 셋업 활성 상태
vk_peak = 당일 VKOSPI 장중 최고치
피크아웃 신호 = VKOSPI가 vk_peak 대비 -5%(초기값) 이상 하락 반전 AND 30분 이상 고점 미갱신
```

### 5.3 통합 지점
- V반등 셋업의 진입 확인 요소로 +1 가점. **단독 진입 트리거로 사용 금지** — L3(방향 형성 확인)의 보조로만

### 5.4 데이터·주의
- VKOSPI 실시간(KRX 산출, 증권사 API 제공). 1분 주기
- VKOSPI는 옵션 호가 기반이라 얇은 장에서 노이즈 큼 → 5분 이동평균으로 평활
- 지수 변동성지수이므로 하닉 개별 공포와 괴리 가능 — 하닉 급락인데 지수는 평온한 날엔 신호 무효

## 6. A1 — ATR 연동 스탑 (R1의 변동성 적응 옵션)

### 6.1 정의와 논리
고정 -3% 스탑(R1)은 평온장 기준으로는 적절하나, 일중 변동폭이 10%를 넘는 고변동장에서는 정상 노이즈에 컷당한다(7/3의 장중 저점 -6.5%가 실례). 스탑 폭을 최근 변동성에 연동해 "노이즈는 견디고 추세 붕괴에는 반응"하도록 조정한다.

### 6.2 계산
```
ATR14 = 14일 평균 진폭 (True Range 평균, 본주 기준 %)
stop_pct = clamp( k × ATR14 × leverage_multiple,  min_stop, max_stop )
  k = 0.7 (초기값, 튜닝 0.5~1.0)
  leverage_multiple = 상품 배수 (2x, 3x)
  min_stop = 3%, max_stop = 8%   # 폭주 방지 상하한

설정 파일:
  stop_mode: "fixed" | "atr"      # 기본 fixed(-3%, 기존 R1 유지)
  trailing_mode: 동일 구조 (기본 -4%, atr 모드 시 k_trail = 0.9)
```

### 6.3 통합 지점
- R1·R2의 계산식 옵션. 알림 메시지에 "오늘의 권장 스탑: -X.X% (ATR 기준)" 병기 — 사용자가 시세포착주문 수치 입력 시 참고

### 6.4 백테스트 (이 모듈의 핵심)
- 비교군: fixed -3% vs ATR k=0.5/0.7/1.0
- 평가지표: ①노이즈 컷 비율(스탑 후 당일 원방향 재개된 거래 비율) ②평균 손실 크기 ③총 손익. **노이즈 컷 감소와 손실 확대는 트레이드오프**이므로 총손익 기준으로 선택

## 7. C1 — 마감 리밸런싱 증폭 (조건부 청산 연기)

### 7.1 정의와 논리
레버리지·인버스 펀드는 일일 배수 유지를 위해 마감 무렵 **당일 지수 방향과 같은 방향**의 리밸런싱 주문을 낸다(지수 상승일엔 매수, 하락일엔 매도 — Cheng & Madhavan 2009의 레버리지 ETF 동학 연구가 이론 기반). 큰 추세일일수록 막판 30분 추세 증폭 경향이 있어, 15:00 일괄 청산(R3)은 추세일의 마지막 구간을 놓칠 수 있다.

### 7.2 규칙 (옵션, 기본 OFF)
```
14:50 판정:
  IF 당일 실시간 DC1 ≥ θ (추세일 확정)
     AND |당일 지수 등락| ≥ 3%
     AND 보유 방향 == 당일 추세 방향
  THEN 청산 시각을 15:00 → 15:15로 연기 (지정가 분할)
  ELSE 기존 R3 유지 (15:00 청산)
```

### 7.3 백테스트 (활성화 전 필수)
- 추세일 표본에서 15:00 / 15:15 / 15:25 청산가 비교
- 실패 모드 확인: 막판 차익실현 물량이 리밸런싱 매수를 압도해 오히려 밀리는 날의 빈도
- 슬리피지 가정: 동시호가 근접 유동성 감소 반영해 청산가에 -0.1% 보수 조정

### 7.4 주의
- 이 효과는 시장 참여자에게 널리 알려져 선반영·소멸됐을 수 있음 — **국내 시장 실데이터 검증 없이는 절대 활성화 금지**

### 8.1 기존 시스템 연결 총괄표

### 8.2 데이터 스키마 확장 (daily_features 테이블 추가 컬럼)

### 8.4 구현 우선순위와 근거
1. **A1 (ATR 스탑)** — 데이터 요구 최소(일봉만), 효과 즉시, 백테스트 단순
2. **N1 (NR7)** — 일봉만으로 구현, 장전 브리핑에 바로 편입
3. **W1 (시장 폭)** — API 한 콜, 왜곡 장세 구분의 실용 가치 높음
4. **B1 (베이시스)** — 20일 축적 필요하므로 수집 로직 먼저 가동
5. **O1 (시가 유형)** — 1분봉 인프라 필요, T-스코어와 시너지 큼
6. **V1 (VKOSPI)** — V반등 셋업 발생일에만 작동하는 저빈도 신호
7. **C1 (마감 증폭)** — 실데이터 검증 전 활성화 금지, 가장 후순위

### 8.5 검증 공통 규약
- 모든 모듈은 활성화 전 **최소 60거래일 기록 → 리프트 검증**(신호 있는 날 vs 없는 날의 목표 지표 차이 유의성) 통과 필수
- 최적화 목표는 상위 문서 8.2 Stage 3과 동일: 적중률이 아니라 거래비용 차감 후 순손익 × 최대낙폭 페널티
- 가점 인플레 방지: 확장 모듈 가점 합산은 T-스코어 총점의 30%를 초과할 수 없음 (핵심 신호가 확장 신호에 묻히는 것 방지)

## 9. RV1 — 하닉 분봉 반전 판정 (사용자 직접 지정, 2026-07-07 — 예외적으로 즉시 활성)

사용자가 직접 정의한 규칙이라 60일 검증 규약의 예외로 **즉시 활성 + 즉시 문자 발송**한다
(단, 하드 블록 우선 원칙은 유지 — 폭락 분기 활성 시 하락 반전의 인버스 문자는 XS1로 차단).

**전제**: 하닉이 직전 추세 중일 것 — 반전 창 시작 이전 30분 순변화 ≥ 0.8%p (방향 무관, 오탐 방지).

**반전 조건** (반대 방향으로 아래 중 하나, 수치는 전일 종가 대비 등락률 차 %p — `config.reversal`):
1. 1분봉 1개 종가 반전 ≥ 0.8%
2. 1분봉 3개 합 ≥ 1.0%
3. 5분봉 1개 ≥ 1.0%
4. 5분봉 3개 합 ≥ 2.2%
5. 5분봉 5개 합 ≥ 2.7%
6. 5분봉 7개 합 ≥ 3.2%

**판정·발송**: 상승 반전 → 레버리지 검토 / 하락 반전 → 인버스 검토. 방향별 1일 1회 문자
(`rev_up`·`rev_down`). 5분봉은 완성된 봉만 사용(진행 봉 진동 방지 — T6과 동일 원칙).
구현: `lib/signal/engine/reversal.ts`(순수) + `lib/signal/alerts.ts` buildReversalAlert.
검증: `scripts/signal-backtest.ts` RV1 섹션. 2026-07-07 실틱 재현 — 09:19 상승 반전
(1분봉 3개 +1.5%p, 직전 -3.8%p)을 실제 반등 시작점에서 포착.
